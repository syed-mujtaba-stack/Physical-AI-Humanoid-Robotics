# Physical AI & Humanoid Robotics Constitution

## Core Principles

### Clarity and Educational Excellence
Content must be beginner-friendly yet technically deep, bridging digital AI to embodied intelligence.

### Modular, Reusable, and AI-Driven Development
Using Spec-Kit Plus and Claude Code for development.

### High Interactivity
Support RAG chatbot, user personalization, and future extensibility.

### Performance
Fast loading, mobile-friendly, accessible.

### Accuracy
All technical details on ROS 2, Gazebo, NVIDIA Isaac Sim, VLA models, hardware requirements must be correct and up-to-date.

### Inclusivity
Support Urdu translation and personalized learning paths.

## Open and Collaborative Development
Fully open-source, well-documented for community contribution.

## Development Workflow
Follows the course outline with Quarter Overview, 4 Modules (ROS 2, Gazebo/Unity, NVIDIA Isaac, Vision-Language-Action), Weekly Breakdown (Weeks 1-13), Capstone Project, Hardware Requirements, Learning Outcomes, and Assessments.

## Governance
All development must align with the core principles; Changes to educational content must maintain technical accuracy; Community contributions must follow established standards.
